<span data-ttu-id="0117d-107">Bir talep tahmini oluşturmak için **Master planlama &gt; Tahmin &gt; Talep tahmini &gt; İstatistik temel tahmin oluştur** menüsüne gidin.</span><span class="sxs-lookup"><span data-stu-id="0117d-107">To generate a demand forecast, go to **Master planning &gt; Forecasting &gt; Demand forecasting &gt; Generate statistical baseline forecast**.</span></span>

<span data-ttu-id="0117d-110">Bir tahminin oluşturulduğu aralıkların sayısı,**Tahmin dönemi** alanında ayarlanır.</span><span class="sxs-lookup"><span data-stu-id="0117d-110">The number of buckets to generate a forecast for is set in the **Forecast horizon** field.</span></span> 

<span data-ttu-id="0117d-111">Tahmin stratejisi **Geçmişteki talep üzerine kopyala** olarak ayarlandığında, geçmiş dönem sonu yok sayılır.</span><span class="sxs-lookup"><span data-stu-id="0117d-111">When the forecast strategy is set to **Copy over historical demand**, the end of the historical horizon is ignored.</span></span> <span data-ttu-id="0117d-112">Sistem, **Tahmin dönemi** alanında belirtilen aralıkların sayısını **Geçmiş dönem** öğesi altındaki **Başlangıç tarihi** alanından başlayarak tahmini talebe kopyalar.</span><span class="sxs-lookup"><span data-stu-id="0117d-112">The system copies the number of buckets specified in the **Forecast horizon** field to the forecast demand, starting from the date set in the **From date** field under **Historical horizon**.</span></span> <span data-ttu-id="0117d-113">Geçmiş talebi belli bir tarihten ileriye doğru kopyalamakla, üretim planlayıcıları sonraki üç aylık dönemin planını iki şekilde yapabilir:</span><span class="sxs-lookup"><span data-stu-id="0117d-113">By copying historical demand from a certain date forward, production planners can make the plan for the next quarter in two ways:</span></span>

-   <span data-ttu-id="0117d-115">Talebi önceki üç aydan başlayarak kopyalamak suretiyle.</span><span class="sxs-lookup"><span data-stu-id="0117d-115">By copying the demand from the previous quarter.</span></span>

<span data-ttu-id="0117d-119">Temel Talep tahmini için başlangıç tarihinin geçerli tarih veya gelecekteki bir tarih olması gerekmez.</span><span class="sxs-lookup"><span data-stu-id="0117d-119">The start date for the baseline demand forecast doesn’t have to be the current date or a date in the future.</span></span> <span data-ttu-id="0117d-120">Farklı bir başlangıç tarihi ayarlamak için, **Temel tahmin Başlangıç tarihi - > [!NOTE]
> <span data-ttu-id="0117d-125">Eksik değer değiştirme yalnızca geçmişe ait verilerin başlangıç ve bitiş tarihleri arasındaki boşluklar için çalışır.</span><span class="sxs-lookup"><span data-stu-id="0117d-125">Missing value substitution only works for the gaps in data in between the start and end dates for historical data.</span></span> <span data-ttu-id="0117d-126">Veri, son fiziksel veri noktasından önce veya sonra doldurmaz, yalnızca gerçek mevcut veri noktaları arasında tahmin edilecek şekilde işlev gösterir.</span><span class="sxs-lookup"><span data-stu-id="0117d-126">It will not fill in data before or after the last physical data point, it only acts as extrapolation between actual existing data points.</span></span>

uygulanabilir.</span><span class="sxs-lookup"><span data-stu-id="0117d-133">Manual adjustments made in previous demand forecasting iterations can be automatically applied to the new baseline forecast ancak silinmezler.</span><span class="sxs-lookup"><span data-stu-id="0117d-134">If the check box is cleared, the manual adjustments are not added to the baseline forecast – but they are not <span data-ttu-id="0117d-139">Talep tahmini oluşturmada, kullanıcıların oluşturulan tahmini tanımalarına yardımcı olacak bir ad ve yorumlar bulunabilir.</span><span class="sxs-lookup"><span data-stu-id="0117d-139">A demand forecast generation can have a name and comments to help users identify the forecast that has been generated.</span></span> <span data-ttu-id="0117d-140">Bu değerler **İstatistik temel tahmin oluşturma geçmişi** sayfasındaki tahmin oluşturma geçmişinde görülür.</span><span class="sxs-lookup"><span data-stu-id="0117d-140">These values are visible in forecast generation history on the **Statistical baseline forecast generation history** page.</span></span> 

<span data-ttu-id="0117d-141">Tahmin oluşturma zamanında şirketlerarası planlama grubu, madde tahsisat anahtarları ve diğer filtreler uygulanabilir.</span><span class="sxs-lookup"><span data-stu-id="0117d-141">The intercompany planning group, item allocation keys, and other filters can be applied at forecast generation time.</span></span> <span data-ttu-id="0117d-142">Bunlar, performansı artırmak veya verileri yönetilebilir parçalara bölmek için kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="0117d-142">These can be used to improve performance or to split the data into manageable chunks.</span></span> <span data-ttu-id="0117d-143">Ancak, sorguda madde tahsisat anahtarı seçilse bile, şirketlerarası planlama grubu ile ilişkili olmayan hiçbir madde tahsisat anahtarının üyeleri için bir talep tahmini oluşturulmayacağını unutmayın.</span><span class="sxs-lookup"><span data-stu-id="0117d-143">However, note that a demand forecast is not generated for the members of any item allocation key that is not associated with an intercompany planning group, even if the item allocation key is selected in the query.</span></span>
