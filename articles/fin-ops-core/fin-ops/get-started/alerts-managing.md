### <a name="frequency-for-change-based-events"></a><span data-ttu-id="43004-116">Değişiklik tabanlı olayların sıklığı</span><span class="sxs-lookup"><span data-stu-id="43004-116">Frequency for change-based events</span></span>

<span data-ttu-id="43004-117">Değişiklik tabanlı olaylar için, sistemin olayı günlüğe kaydetmesinin ardından olayın işlenmesini tetikleyen bir toplu iş oluşturabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="43004-117">For change-based events, you can set up a batch job that triggers the processing of an event soon after the system logs the event.</span></span> <span data-ttu-id="43004-118">Toplu işi, daha sık tekrarlayacak şekilde ayarlarsanız değişiklikler meydana geldiğinde kullanıcılar uyarıları daha erken alır.</span><span class="sxs-lookup"><span data-stu-id="43004-118">If you set up the batch job to recur more often, users receive their alerts sooner after changes occur.</span></span> <span data-ttu-id="43004-119">Ancak daha yüksek toplu işleme sıklığı, sistem performansını olumsuz etkileyebilir.</span><span class="sxs-lookup"><span data-stu-id="43004-119">However, a high frequency for batch processing might adversely affect system performance.</span></span>

<span data-ttu-id="43004-120">Diğer taraftan, daha az tekrarlanan ve sistem yükü düşük olduğu zamanlara planladığınız bir toplu iş, sistem performansını artırmaya yardımcı olabilir.</span><span class="sxs-lookup"><span data-stu-id="43004-120">On the other hand, a batch job that recurs less often, and that you schedule for times when the system load is low, might help improve system performance.</span></span> <span data-ttu-id="43004-121">Ancak, toplu işleme için düşük bir sıklık, kullanıcıların zamanında uyarı taleplerini karşılamayabilir.</span><span class="sxs-lookup"><span data-stu-id="43004-121">However, a low frequency for batch processing might not meet the users' demands for timely alerts.</span></span>

<span data-ttu-id="43004-122">Bu nedenle, değişiklik tabanlı olaylar için toplu işleme sıklığını ayarlarken uyarıların zamanında verilmesi ile tüm sistemin performansı arasındaki dengeyi göz önünde bulundurun.</span><span class="sxs-lookup"><span data-stu-id="43004-122">Therefore, when you set up the frequency of batch processing for change-based events, consider the balance between the timeliness of alerts and the performance of the whole system.</span></span> <span data-ttu-id="43004-123">Uyarı kuralları oluşturan kullanıcıların sayısı arttıkça bu konular daha önemli hale gelir.</span><span class="sxs-lookup"><span data-stu-id="43004-123">These considerations become more relevant as the number of users who create alert rules increases.</span></span> <span data-ttu-id="43004-124">Sıklık, sistemin işlediği olay sayısını etkilemez.</span><span class="sxs-lookup"><span data-stu-id="43004-124">The frequency doesn't affect the number of events that the system processes.</span></span> <span data-ttu-id="43004-125">Ancak daha fazla kullanıcı kural oluşturursa işlem daha fazla denetim çalıştırır.</span><span class="sxs-lookup"><span data-stu-id="43004-125">However, if more users create rules, the process runs more checks.</span></span> <span data-ttu-id="43004-126">Bu tür veri alışverişleri, sistem performansını etkileyebilir.</span><span class="sxs-lookup"><span data-stu-id="43004-126">This type of data exchange might affect system performance.</span></span>

#### <a name="the-risks-of-low-batch-frequency"></a><span data-ttu-id="43004-127">Düşük toplu iş sıklığının riskleri</span><span class="sxs-lookup"><span data-stu-id="43004-127">The risks of low batch frequency</span></span>

<span data-ttu-id="43004-128">Değişiklik tabanlı olayların toplu işlemesi için düşük bir frekans ayarlarsanız uyarı kurallarındaki koşullarla ilgili veriler, işlenmeden önce değişebilir.</span><span class="sxs-lookup"><span data-stu-id="43004-128">If you set up a low frequency for batch processing for change-based events, data that is relevant to the conditions in alert rules might change before processing.</span></span> <span data-ttu-id="43004-129">Bu nedenle, uyarıları kaybedebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="43004-129">Therefore, you might lose alerts.</span></span>

<span data-ttu-id="43004-130">Örneğin, olay **müşteri ilgili kişi bilgileri değişti** ve koşul **müşteri = BB** olduğunda tetiklenmesi için bir uyarı oluşturabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="43004-130">For example, you create an alert to trigger when the event is **customer contact changes** and the condition is **customer = BB**.</span></span> <span data-ttu-id="43004-131">Başka bir ifadeyle, BB müşterisi için müşteri ilgili bilgileri değiştiğinde işlem, olayı günlüğe kaydeder.</span><span class="sxs-lookup"><span data-stu-id="43004-131">In other words, when the customer contact for customer BB changes, the process logs the event.</span></span> <span data-ttu-id="43004-132">Bununla birlikte, toplu işleme sistemi, toplu işlemin veri girişinden daha az sıklıkta gerçekleşeceği şekilde ayarlanır.</span><span class="sxs-lookup"><span data-stu-id="43004-132">However, the batch processing system is set up so that batch processing occurs less often than data entry.</span></span> <span data-ttu-id="43004-133">Müşteri adı, olay işlenmeden önce **BB**'den **AA**'ya değişirse veritabanındaki veri artık kuraldaki koşul (**müşteri = BB**) ile eşleşmez.</span><span class="sxs-lookup"><span data-stu-id="43004-133">If the customer name changes from **BB** to **AA** before the event is processed, the data in the database no longer matches the condition in the rule, **customer = BB**.</span></span> <span data-ttu-id="43004-134">Bu nedenle, olay son olarak işlendiğinde uyarı oluşturulmaz.</span><span class="sxs-lookup"><span data-stu-id="43004-134">Therefore, when the event is finally processed, no alert is generated.</span></span>
