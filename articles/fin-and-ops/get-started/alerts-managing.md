<span data-ttu-id="c1b05-112">Finance and Operations, toplu işlemenin son çalıştırılmasından bu yana gerçekleşen tüm değişiklik tabanlı olayları okur.</span><span class="sxs-lookup"><span data-stu-id="c1b05-112">Finance and Operations reads all change-based events that have occurred since batch processing was last run.</span></span> <span data-ttu-id="c1b05-113">Değişiklik tabanlı olaylar, alanlardaki güncelleştirmeleri, kayıtların silinmesini ve kayıtların oluşturulmasını içerir.</span><span class="sxs-lookup"><span data-stu-id="c1b05-113">Change-based events include updates to fields, the deletion of records, and the creation of records.</span></span> <span data-ttu-id="c1b05-114">Bu olaylar, uyarı kurallarında ayarlanan koşullarla karşılaştırılır.</span><span class="sxs-lookup"><span data-stu-id="c1b05-114">These events are compared with the conditions that are set up in alert rules.</span></span> <span data-ttu-id="c1b05-115">Olay, kuraldaki koşullarla eşleştiğinde toplu işlem bir uyarı oluşturur.</span><span class="sxs-lookup"><span data-stu-id="c1b05-115">When an event matches the conditions in a rule, the batch process generates an alert.</span></span>

### <a name="frequency-for-change-based-events"></a><span data-ttu-id="c1b05-116">Değişiklik tabanlı olayların sıklığı</span><span class="sxs-lookup"><span data-stu-id="c1b05-116">Frequency for change-based events</span></span>

<span data-ttu-id="c1b05-117">Değişiklik tabanlı olaylar için, bir olayın sistem tarafından günlüğe kaydedilmesinin ardından olayın işlenmesini tetikleyen bir toplu iş oluşturabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="c1b05-117">For change-based events, you can set up a batch job that triggers the processing of an event soon after the event is logged by the system.</span></span> <span data-ttu-id="c1b05-118">Toplu işi, daha sık tekrarlayacak şekilde ayarlarsanız değişiklikler meydana geldiğinde kullanıcılar uyarıları daha erken alır.</span><span class="sxs-lookup"><span data-stu-id="c1b05-118">If you set up the batch job to recur more often, users receive their alerts sooner after changes occur.</span></span> <span data-ttu-id="c1b05-119">Ancak daha yüksek toplu işleme sıklığı, sistem performansını olumsuz etkileyebilir.</span><span class="sxs-lookup"><span data-stu-id="c1b05-119">However, a high frequency for batch processing might adversely affect system performance.</span></span>

<span data-ttu-id="c1b05-120">Diğer taraftan, daha az tekrarlanan ve sistem yükü düşük olduğu zamanlara planlanan bir toplu iş, sistem performansını artırmaya yardımcı olabilir.</span><span class="sxs-lookup"><span data-stu-id="c1b05-120">On the other hand, a batch job that recurs less often, and that is scheduled for times when the system load is low, might help improve system performance.</span></span> <span data-ttu-id="c1b05-121">Ancak, toplu işleme için düşük bir sıklık, kullanıcıların zamanında uyarı taleplerini karşılamayabilir.</span><span class="sxs-lookup"><span data-stu-id="c1b05-121">However, a low frequency for batch processing might not meet the users' demands for timely alerts.</span></span>

<span data-ttu-id="c1b05-122">Bu nedenle, değişiklik tabanlı olaylar için toplu işleme sıklığını ayarlarken uyarıların zamanında verilmesi ile tüm sistemin performansı arasındaki dengeyi göz önünde bulundurun.</span><span class="sxs-lookup"><span data-stu-id="c1b05-122">Therefore, when you set up the frequency of batch processing for change-based events, consider the balance between the timeliness of alerts and the performance of the whole system.</span></span> <span data-ttu-id="c1b05-123">Uyarı kuralları oluşturan kullanıcıların sayısı arttıkça bu konular daha önemli hale gelir.</span><span class="sxs-lookup"><span data-stu-id="c1b05-123">These considerations become more relevant as the number of users who create alert rules increases.</span></span> <span data-ttu-id="c1b05-124">Sıklık, işlenmesi gereken olay sayısını etkilemez.</span><span class="sxs-lookup"><span data-stu-id="c1b05-124">The frequency doesn't affect the number of events that must be processed.</span></span> <span data-ttu-id="c1b05-125">Ancak daha fazla kullanıcı kural oluşturursa daha fazla denetim yapılmalıdır.</span><span class="sxs-lookup"><span data-stu-id="c1b05-125">However, if more users create rules, more checks must be done.</span></span> <span data-ttu-id="c1b05-126">Bu tür veri alışverişleri, sistem performansını etkileyebilir.</span><span class="sxs-lookup"><span data-stu-id="c1b05-126">This type of data exchange might affect system performance.</span></span>

#### <a name="the-risks-of-low-batch-frequency"></a><span data-ttu-id="c1b05-127">Düşük toplu iş sıklığının riskleri</span><span class="sxs-lookup"><span data-stu-id="c1b05-127">The risks of low batch frequency</span></span>

<span data-ttu-id="c1b05-128">Değişiklik tabanlı olayların toplu işlemesi için düşük bir frekans ayarlarsanız uyarı kurallarındaki koşullarla ilgili veriler, toplu iş işlenmeden önce değiştirilebilir.</span><span class="sxs-lookup"><span data-stu-id="c1b05-128">If you set up a low frequency for batch processing for change-based events, data that is relevant to the conditions in alert rules might be changed before the batch is processed.</span></span> <span data-ttu-id="c1b05-129">Bu nedenle, uyarıları kaybedebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="c1b05-129">Therefore, you might lose alerts.</span></span>

<span data-ttu-id="c1b05-130">Örneğin, olay **müşteri iletişim bilgileri değişti** ve koşul **müşteri = BB** olduğunda bir uyarıyı tetiklemesi için bir uyarı kuralı ayarlayabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="c1b05-130">For example, an alert rule is set up to trigger an alert when the event is **customer contact changes** and the condition is **customer = BB**.</span></span> <span data-ttu-id="c1b05-131">Başka bir deyişle, BB müşterisi için müşteri iletişim bilgileri değiştirildiğinde olay, günlüğe kaydedilir.</span><span class="sxs-lookup"><span data-stu-id="c1b05-131">In other words, when the customer contact for customer BB is changed, the event is logged.</span></span> <span data-ttu-id="c1b05-132">Bununla birlikte, toplu işleme sistemi, toplu işlemin veri girişinden daha az sıklıkta gerçekleşeceği şekilde ayarlanır.</span><span class="sxs-lookup"><span data-stu-id="c1b05-132">However, the batch processing system is set up so that batch processing occurs less often than data entry.</span></span> <span data-ttu-id="c1b05-133">Müşteri adı, olay işlenmeden önce **BB**'den **AA**'ya değiştirilirse veritabanındaki veri artık kuraldaki koşul (**müşteri = BB**) ile eşleşmez.</span><span class="sxs-lookup"><span data-stu-id="c1b05-133">If the customer name is changed from **BB** to **AA** before the event is processed, the data in the database no longer matches the condition in the rule, **customer = BB**.</span></span> <span data-ttu-id="c1b05-134">Bu nedenle, olay son olarak işlendiğinde uyarı oluşturulmaz.</span><span class="sxs-lookup"><span data-stu-id="c1b05-134">Therefore, when the event is finally processed, no alert is generated.</span></span>
